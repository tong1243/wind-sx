## 4.1 路段运行状态模块

### 4.1.1 全线状态可视化
- 文档章节：`4.1.1`
- 接口：`GET /api/v1/section/visualization/mainline`
- 状态：已实现

请求参数：
| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `timestamp` | long | 是 | 毫秒时间戳 |

返回示例：
```json
{
  "code": 200,
  "success": true,
  "message": "全线状态可视化数据",
  "data": {
    "timestamp": 1711958400000,
    "segmentList": [
      {
        "direction": "哈密",
        "stake": "K3178",
        "startLocation": 0,
        "endLocation": 1000,
        "startStake": "K3178",
        "endStake": "K3179",
        "congestionStatus": "通畅",
        "segmentType": "路段",
        "color": "#00B050",
        "averageSpeed": 86.0
      }
    ]
  }
}
```

字段说明：
- `direction`：方向/幅向
- `stake`：显示桩号
- `startLocation` / `endLocation`：路段起止里程（m）
- `startStake` / `endStake`：路段起止桩号
- `congestionStatus`：`通畅` / `缓行` / `拥堵` / `数据缺失`
- `segmentType`：路段类型（如路段/服务区/互通）
- `color`：渲染色值
- `averageSpeed`：平均速度（km/h）

判定规则（当前实现）：
- `averageSpeed > 80`：通畅（`#00B050`）
- `60 <= averageSpeed <= 80`：缓行（`#FFC000`）
- `averageSpeed < 60`：拥堵（`#C00000`）
- 无速度数据：数据缺失（`#9E9E9E`）

数据来源（当前实现）：
- 静态路段：`road_segment_static`
- 实时断面速度：`section_{date}`（通过 `SectionDataPreService` 聚合）

- 文档章节：`4.1.2`
- 接口：`GET /api/v1/section/detection/parameters`
- 状态：已实现

请求参数：

        "congestionStatus": "通畅",

字段说明：

- `startStake` / `endStake`：路段起止桩号

拥堵判定（当前实现）：

### 4.1.3 事件检测信息
- 文档章节：`4.1.3`
- 接口：`GET /api/v1/event/detection/info`
- 状态：已实现

请求参数：
| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `timestamp` | long | 是 | 毫秒时间戳 |

返回示例：
```json
{
  "code": 200,
  "success": true,
  "message": "事件检测信息数据",
  "data": {
    "timestamp": 1711958400000,
    "recordList": [
      {
        "eventId": "186300001",
        "time": "2026/03/31 10:20:11",
        "direction": "哈密",
        "eventLocation": "K3179+200",
        "eventType": "停车",
        "vehiclePlateNumber": "新A12345",
        "warningStatus": "未处理"
      }
    ]
  }
}
```

字段说明：
- `eventId`：事件ID，优先取 `uuid`，为空时回退 `trajId`
- `time`：事件发生时间
- `direction`：按事件里程匹配 `road_segment_static.direction`
- `eventLocation`：事件桩号位置（如 `K3179+200`）
- `eventType`：`逆行` / `停车` / `低速` / `超速` / `占道` / `其他`
- `vehiclePlateNumber`：涉事车牌
- `warningStatus`：`已处理` / `未处理` / `未知`

状态映射（当前实现）：
- `status == 2`：`已处理`
- 其他状态：`未处理`
- 空值：`未知`

数据来源（当前实现）：
- 事件数据：`carevent_{date}`
- 方向映射：`road_segment_static`

### 4.1.4 服务区进出车辆
- 文档章节：`4.1.4`
- 接口：`GET /api/v1/section/service-area/vehicles`
- 状态：已实现

请求参数：
| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `timestamp` | long | 是 | 毫秒时间戳 |

返回示例：
```json
{
  "code": 200,
  "success": true,
  "message": "服务区进出车辆数据",
  "data": {
    "timestamp": 1711958400000,
    "recordList": [
      {
        "locationName": "红山口服务区南",
        "inCount": 8,
        "outCount": 5,
        "vehicleCount": 27
      },
      {
        "locationName": "红山口服务区北",
        "inCount": 6,
        "outCount": 4,
        "vehicleCount": 21
      }
    ]
  }
}
```

字段说明：
- `locationName`：服务区位置名称
- `inCount`：驶入车辆数（5分钟窗口）
- `outCount`：驶出车辆数（5分钟窗口）
- `vehicleCount`：窗口结束时服务区内车辆数

统计逻辑（当前实现）：
- 时间窗口：`[timestamp - 300000, timestamp]`（5分钟）
- `inCount`：同一 `trajId` 从区间外到区间内记 1 次
- `outCount`：同一 `trajId` 从区间内到区间外记 1 次
- `vehicleCount`：窗口末状态在区间内的唯一 `trajId` 数量
- 服务区区间来源：`road_segment_static` 中 `segment_type` 包含“服务区”或“SERVICE”

数据来源（当前实现）：
- 实时轨迹：`traj_near_real_{date}`
- 服务区区间：`road_segment_static`
